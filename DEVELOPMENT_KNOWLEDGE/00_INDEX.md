# Development Knowledge Base

This directory contains the collective intelligence of the project. It serves as the single source of truth for the Human-AI team.

## Directory Structure

*   **[01_PROJECT_CONTEXT.md](./01_PROJECT_CONTEXT.md)**
    *   *Purpose:* Defines the "What" and "Why".
    *   *Contents:* Tech stack, Architecture, High-level goals.

*   **[02_GUIDELINES.md](./02_GUIDELINES.md)**
    *   *Purpose:* Defines the "How".
    *   *Contents:* Coding standards, Naming conventions, API patterns (DTOs, Contracts), Testing rules.

*   **[03_CURRENT_STATE.md](./03_CURRENT_STATE.md)**
    *   *Purpose:* Defines the "When" and "Where".
    *   *Contents:* Feature status (Ready/Mocked), Technical Debt, Known Bugs.
    *   *Note:* This file supersedes `DEVELOPMENT_STATUS.md`.

*   **[04_DECISION_LOG.md](./04_DECISION_LOG.md)**
    *   *Purpose:* Defines the "History".
    *   *Contents:* Architectural decisions records (ADR), "Memory" of specific constraints (e.g., "Email hash logic").

*   **[05_WORKFLOW_CONTRACT.md](./05_WORKFLOW_CONTRACT.md)**
    *   *Purpose:* Defines the "Who" and "Rules of Engagement".
    *   *Contents:* How the AI and Human interact, Definition of Done, Commit message standards.

*   **[06_PRODUCT_CONTEXT.md](./06_PRODUCT_CONTEXT.md)**
    *   *Purpose:* Defines the "User" and "Product".
    *   *Contents:* User stories, Use cases, Product vision, Functional requirements.
